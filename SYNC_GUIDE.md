## 📱 支持的同步方式

### 方式 1：JSON 导入导出（推荐新手）
**适用场景：**
- 偶尔同步
- 少量数据
- 不想配置 GitHub

**操作步骤：**

#### 电脑端导出数据
1. 进入"系统管理"页面
2. 点击"📤 导出数据"
3. 保存 JSON 文件到本地
4. 通过微信/QQ/邮件发送到手机

#### 手机端导入数据
1. 接收 JSON 文件
2. 用文本编辑器打开
3. 复制全部内容
4. 进入"系统管理" → "数据同步"
5. 选择"2 - 导入数据"
6. 粘贴 JSON 内容
7. 刷新页面

**优点：** 简单直接，无需配置  
**缺点：** 需要手动操作，容易出错

---

### 方式 2：GitHub 同步（推荐）
**适用场景：**
- 频繁同步
- 多设备使用
- 自动备份

**准备工作：**

#### 1. 获取 GitHub Token
1. 访问 https://github.com/settings/tokens
2. 点击"Generate new token"
3. 填写备注（如：Clinic Sync）
4. 勾选权限：`repo`（完整仓库权限）
5. 点击"Generate token"
6. **立即复制 Token**（只显示一次）

#### 2. 首次同步

##### 电脑端上传数据
1. 进入"系统管理"页面
2. 点击"🔄 数据同步"
3. 选择"3 - 同步到 GitHub"
4. 粘贴 GitHub Token
5. 等待同步成功提示

##### 手机端下载数据
1. 进入"系统管理"页面
2. 点击"🔄 数据同步"
3. 选择"4 - 从 GitHub 下载"
4. 粘贴相同的 GitHub Token
5. 等待下载成功提示
6. 刷新页面

**优点：** 自动同步，一次配置永久使用  
**缺点：** 需要 GitHub 账号，首次配置稍复杂

## ⚠️ 注意事项

### 数据安全
- ✅ GitHub Token 妥善保管，不要分享
- ✅ 定期本地备份 JSON 文件
- ✅ 使用私密网络同步

### 数据冲突
- ⚠️ 避免同时在两个设备修改同一患者
- ⚠️ 同步前先确认数据版本
- ⚠️ 如有冲突，以最新数据为准

### 同步限制
- GitHub 仓库大小限制：1GB
- 单次提交大小限制：100MB
- 建议定期清理旧数据

## 🐛 常见问题

### Q1: Token 无效？
**A:** 检查 Token 是否正确复制，确保勾选了 `repo` 权限

### Q2: 同步失败？
**A:** 检查网络连接，确认 GitHub 仓库存在且有写入权限

### Q3: 数据丢失？
**A:** 立即从 GitHub 下载或从 JSON 备份恢复

### Q4: 如何重置数据？
**A:** 进入"系统管理" → "清空所有数据"，然后重新导入
